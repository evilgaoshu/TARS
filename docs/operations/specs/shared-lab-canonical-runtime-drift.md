# Shared Lab Canonical Runtime Drift Spec

Issue: EVI-18

## Goal

Keep the shared TARS lab on `192.168.3.100` running from the canonical path `/data/tars-setup-lab`, and make any runtime identity drift visible before PR or QA acceptance.

## Canonical Runtime Contract

The default canonical runtime root is `/data/tars-setup-lab`.

Required paths:

- Binary: `/data/tars-setup-lab/bin/tars-linux-<arch>-dev`
- Working directory: `/data/tars-setup-lab/team-shared` or another child of `/data/tars-setup-lab`
- Shared env: `/data/tars-setup-lab/team-shared/shared-test.env`
- Runtime metadata: `/data/tars-setup-lab/team-shared/runtime_git_head`
- Managed service: `tars-shared-lab.service`

The shared env must include `TARS_DIR=/data/tars-setup-lab`. Runtime `TARS_*` path variables must also resolve inside the canonical root.

## Owner-Accepted Exception

Non-canonical runtime paths are not accepted unless the owner records an exception in an issue comment or through `/data/tars-setup-lab/.canonical-override`.

The `.canonical-override` file uses shell-style `key=value` lines and must contain:

```text
accepted_path=/absolute/non-canonical/path
owner_id=<owner-id-or-name>
date=YYYY-MM-DD
reason=<why the exception is accepted>
expires_at=YYYY-MM-DD
```

The maximum intended lifetime is 90 days. Missing fields, relative paths, invalid dates, or expired dates make the check fail. A valid exception converts matching non-canonical path checks from FAIL to WARN and prints the accepted path, owner, date, reason, and expiry.

## Verification Requirements

`scripts/check-shared-lab.sh` is the shared-lab runtime identity gate. It must verify:

- Listener pid for port `8081`
- Binary path
- Process cwd
- Process `TARS_DIR`, `TARS_HOME`, and `TARS_*` path env values
- `shared-test.env` location and `TARS_DIR`
- Deployed runtime git head against `TARS_SHARED_LAB_EXPECTED_GIT_HEAD`
- Managed systemd unit `WorkingDirectory`, `ExecStart`, and `EnvironmentFile` or `Environment`
- Auth login, setup/status, and optional session URL reachability

Any mismatch outside the canonical root must fail unless a valid owner-accepted exception covers the actual path.

## Deployment Requirements

Shared-host deploys to `192.168.3.100` default to `/data/tars-setup-lab`. The deployment flow writes `team-shared/runtime_git_head` with the source commit used for the deployed binary.

Runtime restart must use the managed systemd unit generated by `scripts/lib/shared_remote_service.sh`. Future agents must not start the shared runtime with a bare binary, `nohup`, or background ampersand.

## Evidence Requirements

Each EVI-18 delivery must commit a verification record under `docs/operations/records/`. The record must include:

- Final canonical path or owner-accepted exception path
- pid, binary, cwd, service unit, and shared env path
- runtime git head and PR/head commit relationship
- commands executed on `192.168.3.100`
- PASS/FAIL summary, blockers, and log locations
